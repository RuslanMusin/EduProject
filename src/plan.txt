Структура
1) Граф обучения (EduGraph)
- Все Предметы
- Пройденные предметы
- Предметы, которые можно начать изучать.

2) Предмет (Subject)
- Название
- Список требований от обр.системы.
- Список предметов, которые нужно пройти для того, чтобы пройти данный предмет.

4) Студент (модель студента)
- Имя
- Семестр обучения
- Граф обучения

7) Рынок
- содержит работы

8) Работа
- название
- имеет требования (Requirement)
- востребованность (среднее кол-во предложений)

9) Требование
- название
- какой предмет (то есть какой образовательный предмет содержит данное требование)

===================
Есть три стейкхолдера: рынок, государство, сам студент.
Проблема: планирование траектории развития студента с учетом
разных требований стейкхолдеров. Разные требования, нужно
постараться выстроить путь развития, учитывая все требования.
Цель: создание программу, в которой будут учитываться
оценки, желания (интерес) и выполненные требования (прохождение каждого предмета приводит
к выполнению определенных требований, требования - почти что навыки + знания) студента,
требования рынка (кол-во вакансий на работу) и требования гос-ва к предметам.
Исходя из этих требований будут предлагаться рекомендации по
построению траектории развития.
Кому это нужно: в первую очередь студенту, потом работадателям,
потом государству. Государство не предъявляет какой-то особый
образ студента на выходе (по требованиям это всезнайка по чуть-чуть),
рынок конкретизирует образ сотрудника, образ выпускника для самого студента
также во многом совпадает с рынком.
Как это примерно работает:
Есть студент Руслан.
Допустим, он на первом курсе. В начале по идее
траектория пуста, начинает работать она преимущественно
после 1-го семетра (когда уже есть оценки и хотя бы какое-то
понимание своих интересов у студента). Вот первый семестр пройден.
Руслан прошел следующие предметы:
Линейная алгебра (72)
Дискретная математика (69)
Мат. анализ (82)
Баллы уже есть. Интерес можно расставить так: алгебра, дискретка,
анализ (присваивается самим студентом оценка от 1 до 10).
Потом после прохождения курсов студент получает
новые навыки (знания, умения), то есть при успешной
сдаче он получает галочку по образовательному стандарту по этому
предмету. Допустим, для линейной алгебры это
могут быть (знание умножения матриц, работа с комплексными числами).
Далее есть требование рынка: есть следующие работы
- Разработчик игр (5000)
- Javascript разработчик (11000)
- Android разработчик (4000)
В скобках указана востребованность по HeadHunter например. У каждой работы свои требования
Допустим, Android разработчик:
1) Знание основных алгоритмов и структур
2) Знание Kotlin
3) паттерн MVP
4) Unit тестирование
5) Android SDK
Эти требования покрываются несколькими предметами
1) Android (нужна информатика)
2) Тестирование
3) Алгоритмы и структуры (нужна информатика)
У предметов тоже есть требования.
По итогу как выходит. Исходя из данных студента прошлой
траектории развития и рыночных требований студенту
может быть предложена Android разработка вместо
Разработки Сайтов или Игр. Так его путь будет постепенно
выстраиваться в нужную сторону.
Вообще, вся суть больше видна конечно именно когда есть курсы
по выбору.

=====Суть====
Допустим студент прошел вот эти курсы (первые три). Далее по пути у него невыборный предмет - алгоритмы. Нужно знать все три предмета.
После этого предмета уже есть выбор: Основы GameDevelop, Разработка дин.сайтов, Android разработка.
Для GameDev требуются знания всех трех дисциплин, для Javascript только лин.алгебра, для
Android - лин.алгебра и дискретка. Для всех нужны алгоритмы.
Как будет предлагаться рекомендация?
Нужна формула:
value = норм. предполагаемый интерес
        + норм.оценка по предметам, которые нужны для прохождения предмета
        + норм. интерес по прошлым предметам
        + норм. кол-во вакансий некоторых работ (у которых требования пересекаются с предметом)
